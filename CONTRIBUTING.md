# Contributing to QA Prompt Library

Thank you for your interest in contributing to the QA Prompt Library! This document provides guidelines for contributing to this project.

## How to Contribute

### 1. Fork the Repository
- Click the "Fork" button at the top right of the repository page
- Clone your forked repository to your local machine

### 2. Create a Branch
```bash
git checkout -b feature/your-prompt-name
```

### 3. Make Your Changes
- Add your prompts to the appropriate section in `QA_PROMPT_LIBRARY.md`
- Follow the existing format and structure
- Ensure your prompts are clear, practical, and well-documented

### 4. Test Your Prompts
- Test your prompts with AI tools (ChatGPT, Claude, etc.)
- Verify they produce useful, accurate results
- Include examples of expected outputs

### 5. Commit Your Changes
```bash
git add .
git commit -m "Add: [Brief description of your contribution]"
```

### 6. Push to Your Fork
```bash
git push origin feature/your-prompt-name
```

### 7. Submit a Pull Request
- Go to the original repository
- Click "New Pull Request"
- Select your branch
- Provide a clear description of your changes

## Contribution Guidelines

### Prompt Quality Standards
- **Clarity**: Prompts should be clear and unambiguous
- **Completeness**: Include all necessary context and parameters
- **Practicality**: Focus on real-world QA scenarios
- **Examples**: Provide concrete examples with sample data
- **Format**: Follow the existing template structure

### What to Include in Your Prompt
1. **Use Case Title**: Clear, descriptive title
2. **Prompt Template**: Generic template with placeholders
3. **Example**: Filled-in example with realistic data
4. **Context**: When and why to use this prompt
5. **Expected Output**: What kind of results to expect

### Prompt Template Format
```markdown
#### Use Case X: [Title]
**Prompt:**
```
[Generic prompt template with [placeholders]]
```

**Example:**
```
[Filled example with actual values]
```
```

### Categories for New Prompts
Add your prompts to the most relevant category:
- Test Case Design
- Test Automation
- Bug Analysis & Reporting
- Test Planning & Strategy
- Performance & Load Testing
- Security Testing
- Mobile Testing
- Data Testing
- Accessibility Testing
- Test Data Management
- Regression Testing
- Exploratory Testing
- CI/CD & DevOps
- Test Metrics & Reporting

If your prompt doesn't fit existing categories, suggest a new category in your PR description.

## Code of Conduct

### Our Standards
- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on constructive feedback
- Respect differing viewpoints and experiences

### Unacceptable Behavior
- Harassment or discriminatory language
- Trolling or insulting comments
- Personal or political attacks
- Publishing others' private information

## Review Process

1. **Initial Review**: Maintainers will review your PR within 3-5 business days
2. **Feedback**: You may receive feedback or change requests
3. **Iteration**: Make requested changes and update your PR
4. **Approval**: Once approved, your PR will be merged
5. **Recognition**: Contributors will be acknowledged in the README

## Types of Contributions

### High Priority
- New prompt templates for common QA scenarios
- Examples for existing prompts
- Corrections to existing prompts
- Documentation improvements

### Welcome Contributions
- Additional use cases
- Tool-specific variations (Selenium, Playwright, etc.)
- Language-specific examples (Java, Python, JavaScript)
- Industry-specific prompts (healthcare, finance, etc.)

### Ideas for Contribution
- Prompts for emerging testing areas (AI/ML testing, blockchain testing)
- Integration with specific tools (Jira, TestRail, etc.)
- Video tutorials or guides
- Translations to other languages

## Questions or Suggestions?

- **Issues**: Open an issue for bugs or feature requests
- **Discussions**: Use GitHub Discussions for questions and ideas
- **Contact**: Reach out to maintainers through GitHub

## Recognition

All contributors will be recognized in the project. Significant contributions may earn you:
- Mention in the README
- Contributor badge
- Co-maintainer status (for ongoing contributors)
